Create a complete API integration auth system for the ecommerce website so the ERP can connect by copy-and-paste only.

Goal:
The website must generate and store its own API token, expose a working health/test endpoint, and provide a visual admin page showing:
- Base API URL
- API token
- Copy buttons

- Test endpoint status

Important:
Do not remove existing routes.
Only add the new integration auth layer and UI.

========================================
1) BACKEND REQUIREMENTS
========================================

Implement these endpoints in the site API:

GET /api/health
Response 200:
{ "status": "ok", "service": "valfre ecommerce api" }

GET /api/test
Response 200:
{ "status": "connected", "service": "valfre ecommerce api" }

GET /api/ping
Response 200:
{ "status": "connected", "service": "valfre ecommerce api" }

Also implement protected integration endpoints using Bearer token auth.

Create these new endpoints:

GET /api/integration/config
Returns:
{
  "baseUrl": "https://YOUR-SITE-DOMAIN/api",
  "tokenPreview": "vlf_site_xxxxx",
  "tokenExists": true
}

POST /api/integration/generate-token
Generates a secure token if none exists yet.
Response:
{
  "success": true,
  "token": "FULL_TOKEN_HERE",
  "baseUrl": "https://YOUR-SITE-DOMAIN/api"
}

POST /api/integration/regenerate-token
Invalidates old token and creates a new one.
Response:
{
  "success": true,
  "token": "NEW_FULL_TOKEN_HERE",
  "baseUrl": "https://YOUR-SITE-DOMAIN/api"
}

POST /api/integration/validate
Headers:
Authorization: Bearer <token>
Response 200:
{
  "success": true,
  "authorized": true,
  "service": "valfre ecommerce api"
}

GET /api/integration/site-info
Headers:
Authorization: Bearer <token>
Response:
{
  "success": true,
  "siteName": "Ferramentas Valfre - Loja Online",
  "apiVersion": "1.0.0",
  "features": {
    "orders": true,
    "products": true,
    "stock": true,
    "coupons": true,
    "shipping": true,
    "abandonedCart": true
  }
}

Auth rules:
- Token must be stored securely in database or environment-backed persistent storage
- Token must NOT be hardcoded
- Use Bearer token authentication
- Return 401 for invalid token
- Add helper middleware requireSiteIntegrationAuth()

========================================
2) TOKEN FORMAT
========================================

Generate secure token in this style:
vlf_site_<random_long_secure_string>

Example:
vlf_site_9a7f3c4e8d91ab34f2c6e8d5f712c901

Use crypto-secure random generation.
