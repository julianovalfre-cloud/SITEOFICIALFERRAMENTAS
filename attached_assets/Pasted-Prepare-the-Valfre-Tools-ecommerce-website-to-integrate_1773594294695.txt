Prepare the Valfre Tools ecommerce website to integrate with a future ERP backend.

IMPORTANT RULES

The website must NOT integrate directly with:
- Tiny ERP
- Melhor Envio
- SEFAZ
- internal fiscal logic

All integrations must go through a future ERP API.

Architecture that must be respected:

Website
↓
ERP API
↓
Tiny ERP
Mercado Pago
Melhor Envio
SEFAZ

The website will only consume ERP endpoints.

Do not create debug loops.
Do not modify the current Mercado Pago checkout integration.
Do not break the current UI.

This task only prepares the website for ERP integration.

--------------------------------------------------

PART 1 — CREATE ERP API CLIENT

Create a centralized API client module for ERP communication.

Example file:

services/erpApi.ts

This client must read environment variable:

ERP_API_URL

Example functions:

getProducts()
getProductBySlug(slug)
getStock(sku)
calculateShipping(payload)
createOrder(orderPayload)
validateCoupon(code)
getOrder(orderId)
getTracking(orderId)

Use fetch with JSON responses.

PART 3 — STOCK CHECK FROM ERP

Before adding to cart, verify stock through ERP endpoint.

ERP endpoint that will exist later:

GET /api/store/stock/:sku

Example response:

{
 "sku":"MAR-001",
 "stock":12,
 "available":true
}

If stock is unavailable, disable Add to Cart button.

--------------------------------------------------

PART 4 — SHIPPING CALCULATION VIA ERP

Implement shipping calculation UI but call ERP endpoint.

Create function:

calculateShipping(cep, cartItems)

POST /api/store/shipping/calculate

{

 "products":[
  {
   "sku":"MAR-001",
   "name":"Martelo Tramontina",
   "width":20,
   "height":10,
   "length":25,
   "weight":0.5,
   "insurance_value":47.90,
   "quantity":1
  }
 ]
}

Display returned options:

Carrier
Price
Delivery time

Include:

Correios

Selected shipping must update order summary.
